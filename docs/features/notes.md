# Notes

Project-scoped Markdown notes with task lists, pop-out windows, and quick capture from audits and sessions.

## Location

Notes live in **`<project>/Notes/`**. On first use, VibeBar offers setup including an optional `.gitignore` entry for the folder.

## Using the panel

1. Open **Notes** from the toolbar or palette.
2. Create or select a note (Markdown).
3. Edit with standard Markdown — **task lists** (`- [ ]`) show checklist progress in the sidebar.
4. **Pop out** a note into a sticky always-on-top window for reference while coding.

## Save to note

From other panels:

- **Security Audit** findings → save summary to a note
- **Session Hub** / terminal issues → append to a note

Useful for tracking fix backlog or PR checklists.

## Tips

- Name notes by sprint, feature, or audit date for easy handoff.
- Pop-out windows persist bounds like other floating VibeBar windows.
- Notes are plain files — edit them outside VibeBar anytime.
